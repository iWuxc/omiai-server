# 前端技术选型评估报告

## 1. 候选方案对比

本次评估主要对比 **微信原生开发 (Native)** 与 **UniApp (Cross-platform)** 两种主流方案。

| 维度 | 微信原生开发 (Native) | UniApp (Vue.js based) |
| :--- | :--- | :--- |
| **开发语言** | WXML, WXSS, JavaScript/TypeScript | Vue.js (SFC), JavaScript/TypeScript |
| **学习成本** | 中（需学习微信特定语法） | 低（Vue生态成熟，开发者众多） |
| **跨平台能力** | 无（仅限微信） | **强**（一套代码可发布至微信、抖音、H5、App） |
| **开发效率** | 一般（组件库需自行集成或使用WeUI） | **高**（内置丰富组件，插件市场庞大） |
| **性能表现** | ⭐⭐⭐⭐⭐ (极致) | ⭐⭐⭐⭐ (优秀，仅有微小损耗) |
| **生态/社区** | 官方文档最全 | 社区活跃，uView等UI库非常成熟 |
| **维护成本** | 较高（若未来做App需重写） | 低（多端复用） |

## 2. 详细分析

### 2.1 微信原生开发
- **优势**：
    - 直接调用微信底层API，无兼容性问题。
    - 性能理论最优，包体积最小。
    - 调试报错最直接，官方支持响应快。
- **劣势**：
    - 代码无法复用到其他平台（如未来想做Web版管理后台）。
    - 样式写法 (WXSS) 与标准 CSS 有细微差异，开发体验不如 SCSS/Less 灵活。

### 2.2 UniApp 开发
- **优势**：
    - **"一次编写，多端运行"**：您提到“期望载体是微信”，但作为一个SaaS应用，未来极有可能需要 **Web端管理后台**（方便电脑操作8000+数据）。UniApp 可以直接将小程序代码编译为 H5 网页，极大节省成本。
    - **开发速度快**：配合 `uView UI` 或 `Uni-ui`，可以像搭积木一样快速构建漂亮的表单和列表。
    - **Vue 语法**：逻辑层与视图层分离更清晰，状态管理 (Vuex/Pinia) 更适合复杂的数据流（如多条件筛选）。

## 3. 最终决策建议

**推荐方案：UniApp**

### 决策依据：
1.  **未来扩展性**：红娘管理8000+客户，在手机上操作虽然便捷，但未来很大概率需要 **PC Web端** 进行批量导入、导出或深度分析。UniApp 原生支持编译为 H5，可低成本迁移。
2.  **开发效率**：UniApp 生态中有大量现成的“表单模板”和“筛选组件”，非常契合本项目“信息收集+筛选”的核心需求。
3.  **人才储备**：Vue.js 是目前最流行的前端框架之一，后续维护或招人更容易。

## 4. 开发规范 (UniApp)
- **框架版本**：Vue 3 + Vite
- **UI 库**：`uView UI 2.0` (功能最全) 或 `Wot Design Uni` (轻量现代化)
- **CSS 预处理**：SCSS
- **网络请求**：`luch-request` (拦截器封装完善)
