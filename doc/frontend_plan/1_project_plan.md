# 红娘助手小程序 - 前端开发计划书

## 1. 项目目标
构建一款基于微信生态的“红娘助手”小程序，解决8000+客户的管理、筛选、匹配及信息收集痛点。
- **核心价值**：提高红娘管理效率，降低人工录入成本（用户自填），提升匹配精准度。
- **目标载体**：微信小程序（红娘管理端 + 客户填单端）。

## 2. 功能需求清单

### 2.1 红娘管理端 (Admin Side)
1.  **工作台 (Dashboard)**
    *   今日新增客户数、待处理匹配、今日待办。
    *   快捷入口：录入客户、分享邀请链接。
2.  **客户公海池 (Client Pool)**
    *   **列表展示**：卡片式展示客户摘要（头像、姓名、年龄、学历、收入）。
    *   **超级筛选**：支持多维度交叉筛选（年龄段、身高、学历、收入、房车等）。
    *   **搜索**：按姓名、手机号精准查找。
3.  **客户详情 (Profile Detail)**
    *   展示所有基础信息及择偶要求。
    *   **操作栏**：编辑、删除、**生成匹配**、分享名片。
4.  **智能匹配 (Matching)**
    *   查看某位客户的匹配结果列表。
    *   筛选匹配结果。

### 2.2 客户填单端 (User Side)
1.  **信息采集表单 (Registration Form)**
    *   分步填写：基础信息 -> 详细画像 -> 择偶标准 -> 照片上传。
    *   自动保存草稿（防止填写中断）。
2.  **授权登录**
    *   微信一键手机号授权（可选，用于绑定身份）。
3.  **提交成功页**
    *   提示已提交，展示红娘联系方式。

## 4. 团队分工与配置
- **产品/UI**：负责原型确认与视觉设计（User）。
- **后端开发**：提供API接口（已就绪，Based on Go）。
- **前端开发**：负责小程序页面与逻辑（AI Assistant）。

## 5. 风险评估与预案
1.  **风险**：微信审核不通过（涉及社交/婚恋类目）。
    *   *预案*：提前准备“生活服务-婚庆/交友”类目资质，或调整功能描述为“CRM内部管理”。
2.  **风险**：用户上传图片违规。
    *   *预案*：接入微信内容安全API（img_sec_check）。
3.  **风险**：表单过长导致用户流失。
    *   *预案*：设计为“分步保存”模式，并增加进度条激励。
